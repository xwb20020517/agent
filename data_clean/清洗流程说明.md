# 数据清洗流程说明

本文档用于复习 `data_clean` 目录下的清洗流程。当前流程分为三步：

1. `clean_1.py`：从 MinerU 原始 JSON 中保留页级核心字段。
2. `clean_2.py`：清洗 `para_blocks`，生成适合文本 RAG 的结构，同时单独生成图片清单。
3. `clean_3.py`：把 `cleaned_2.json` 转换成最终入库用的 `chunks.jsonl`。

## 目录文件

| 文件 | 作用 |
| --- | --- |
| `MinerU_row.json` | MinerU 原始解析结果。 |
| `clean_1.py` | 第一步清洗脚本。 |
| `cleaned_1.json` | 第一步输出。 |
| `clean_2.py` | 第二步清洗脚本。 |
| `cleaned_2.json` | 第二步输出，文本 RAG 的结构化内容。 |
| `image_manifest.json` | 第二步输出，图片信息清单，不进入文本 RAG。 |
| `clean_3.py` | 第三步 chunk 脚本。 |
| `chunks.jsonl` | 第三步输出，最终 RAG chunks。 |

## 第一步：页级清洗

脚本：`clean_1.py`

目标：

- 读取 MinerU 原始 JSON。
- 遍历 `pdf_info`。
- 每页只保留：
  - `page_idx`
  - `page_number`
  - `para_blocks`
- 从 `discarded_blocks` 中提取 `type == page_number` 的页码。

运行示例：

```powershell
python -c "from clean_1 import clean_json_file; clean_json_file('MinerU_row.json','cleaned_1.json','clean_1_errors.json')"
```

输出：

```text
cleaned_1.json
```

## 第二步：para_blocks 清洗

脚本：`clean_2.py`

目标：

- 去掉 MinerU 里不适合 RAG 的版面字段，例如 `bbox`、`angle`、`score` 等。
- 提取 `title`、`text`、`list`、`table` 的核心语义内容。
- 图片不进入文本 RAG，单独写入 `image_manifest.json`。

运行示例：

```powershell
python clean_2.py cleaned_1.json cleaned_2.json
```

### 文本块

`title` 和 `text` 会被整理成：

```json
{
  "type": "text",
  "content": "正文内容",
  "index": 10
}
```

`title` 会额外保留 `level`：

```json
{
  "type": "title",
  "content": "充电说明",
  "level": 2,
  "index": 2
}
```

### 列表块

`list` 会被整理成：

```json
{
  "type": "list",
  "content": "- 第一条\n- 第二条",
  "items": ["第一条", "第二条"],
  "index": 4
}
```

### 表格块

表格会单独结构化，不和正文混在一起。

输出结构：

```json
{
  "block_idx": 22,
  "type": "table",
  "caption": "一般充电故障诊断",
  "content": "表格：一般充电故障诊断\n\n第1条：\n故障状态：不能充电...\n可能原因：...\n解决方法：...",
  "markdown": "| 故障状态 | 可能原因 | 解决方法 |\n| --- | --- | --- |",
  "rows": [
    {
      "故障状态": "不能充电:物理连接完成,已启动充电",
      "可能原因": "充电卡欠费或充电桩故障",
      "解决方法": "充电卡费用查询或联系场站工作人员解决。"
    }
  ]
}
```

表格清洗规则：

- 从 HTML table 中解析表头和行。
- 处理 `rowspan`、`colspan`。
- 对第一列这类上下文列做向下填充，避免出现空的“故障状态”。
- `content` 只放 RAG 检索友好的逐条文本。
- `markdown` 只用于保留可读表格，不混入 `content`，避免重复。
- `rows` 用字典数组保存结构化表格行。

### 图片清单

图片不进入 `cleaned_2.json` 的文本 RAG 内容。

真正的 `type == image` 图片块会进入：

```text
image_manifest.json
```

保留字段包括：

- `page_idx`
- `page_number`
- `para_idx`
- `block_idx`
- `source_block_type`
- `bbox`
- `image_refs`
- `content_descriptions`

说明：

- 表格截图不进入 `image_manifest.json`，因为表格已经被文本化。
- `nearby_text` 默认不保留，避免把普通正文重复混入图片清单。
- 后续如果要做多模态 RAG，可以通过 `page_idx`、`para_idx`、`bbox` 回溯上下文。

## 第三步：生成 chunks.jsonl

脚本：`clean_3.py`

目标：

- 把 `cleaned_2.json` 转换成最终 RAG 入库用的 `chunks.jsonl`。
- 每行一个 JSON chunk。

运行示例：

```powershell
python clean_3.py cleaned_2.json chunks.jsonl --source-file xia.pdf
```

如果不传 `--source-file`，默认会使用输入文件名推断，例如 `cleaned_2.pdf`。

## Chunk 切分规则

### 正文 chunk

规则：

- 以 `title` 作为正文 chunk 边界。
- 一个 `title` 到下一个 `title` 前的 `text/list` 合成一个正文 chunk。
- `title` 本身进入 `content`。
- `text/list` 不决定边界，只进入当前 chunk。
- `chunk_type = text`。

示例：

```json
{
  "chunk_type": "text",
  "section_title": "充电安全警告",
  "content": "充电安全警告\n\n- 禁止未成年人进行充电作业..."
}
```

### 表格 chunk

规则：

- `table` 单独成 chunk。
- 不并入正文 chunk。
- `chunk_type = table`。
- `section_title` 使用表格前面最近的标题。
- 表格 chunk 保留：
  - `content`
  - `rows`
  - `markdown`
  - `caption`
  - `block_idx`

## 长度兜底规则

### 普通正文

- `80~800` 字：直接保留。
- `<80` 字：如果语义完整，直接保留。
- `<80` 字且只有标题：尝试和下一个非 table chunk 合并。
- `>800` 字：递归文本分割。
- 切开后的子 chunk 保留原 `section_title`。

递归分割优先级：

1. 空行
2. 换行
3. 句号
4. 分号
5. 逗号
6. 空格
7. 硬切

### 表格

- 不管多短，都单独保留。
- `<800` 字：直接作为一个 table chunk。
- `>800` 字：按 `rows` 分批切分。
- 每个表格子 chunk 都保留表头字段和值。
- 合并单元格继承已经在第二步完成。

## 最终 chunk 格式

每一行 JSONL 的基本格式：

```json
{
  "chunk_id": "xia_p12_c003",
  "source_file": "xia.pdf",
  "page_idx_start": 12,
  "page_idx_end": 13,
  "page_number_start": "10",
  "page_number_end": "11",
  "chunk_type": "text",
  "section_title": "车辆启动",
  "content": "车辆启动。启动车辆前请确认钥匙在车内……",
  "metadata": {
    "source_file": "xia.pdf",
    "page_idx_start": 12,
    "page_idx_end": 13,
    "page_number_start": "10",
    "page_number_end": "11",
    "chunk_type": "text",
    "section_title": "车辆启动"
  }
}
```

表格 chunk 会额外包含：

```json
{
  "block_idx": 22,
  "caption": "一般充电故障诊断",
  "rows": [],
  "markdown": "| 故障状态 | 可能原因 | 解决方法 |"
}
```

## 推荐完整运行顺序

```powershell

python clean_2.py cleaned_1.json cleaned_2.json
python clean_3.py cleaned_2.json chunks.jsonl --source-file xia.pdf
```
